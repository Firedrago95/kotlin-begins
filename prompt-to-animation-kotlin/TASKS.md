## 📅 Day 1: 레거시 구현 실수 분석 및 코틀린 프로젝트 환경 구성

### 1. 레거시(Java) 구현 실수 및 보완점 분석 완료
과거 진행했던 과제전형 코드를 리뷰하여 코틀린 전환 시 보완해야 할 5가지 치명적 실수를 파악했습니다.
1. **멱등키 `orElseGet()` 사용 실수**: DB Unique Index 없이 레이스 컨디션 발생 시 중복 데이터가 삽입되거나 예외 발생 위험.
2. **웹훅 + 폴링 중복 실행**: 컷 상태 검증 로직 누락으로 비디오 생성 API 중복 호출 위험.
3. **중복 실행 시 클라이언트 에러 노출**: 낙관적 락(`@Version`) 예외를 삼키지 않고 500 에러로 클라이언트에 전파.
4. **멱등성 키 무용지물 (OpenAI 중복 호출)**: 기존 프롬프트를 조회해놓고도 무조건 OpenAI API를 다시 호출하여 비용 낭비.
5. **지연된 웹훅 예외 남발**: 이미 처리/삭제된 건에 대해 뒤늦게 웹훅이 도착하면 단순 무시해야 하나 `orElseThrow`로 불필요한 서버 에러 발생.

### 2. 코틀린 단일 서버 프로젝트 세팅 완료
- `prompt-to-animation-kotlin` 폴더 내에 프론트엔드를 제외한 백엔드 전용 프로젝트 구성.
- Spring Boot 3.3.x, Kotlin 1.9.x, Gradle(KTS) 기반 단일 모듈 환경 구축.
